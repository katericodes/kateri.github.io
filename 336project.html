Auction Website

Context and Objectives

Auction websites are provide an online marketplace for users to sell and buy products with ease. In this project, my group and I were tasked to design and implement a relational database system to support the operations of an create an online auction site. Customers would have the ability to create accounts, login/logout, list items for sale, search for and buy items through bids, and create a wishlist for items not yet on sale.

Approach

For our auction website, we created an auction website for electronics

The website was created using HTML, MySQL, and Javascript and hosted locally. HMTL was used for the user interface, MySQL was used for the database server, and Javascript was used for connectivity between the two. A starter code with example HTML and Javascript code was provided for students to focus more on the SQL database and queries.

Within the group, I contributed to the following:

  • design of the ER diagram and Relational Schema,
  • account creation and login/logout functionality, and
  • search queries related to browsing and search functionality.

Acknowledgments

This project was completed with a group for the CS336 Course taught at Rutgers University. Only a sample of the project with respect to my contributions is published to comply with Rutgers Academic Integrity.

Database Design and Entity Relationship (ER) Diagram

The auction website is for electronics that were further categorized into desktops, laptops, and tablets. These electronics had the following attributes: product ID, model, brand, operating system, screen size, and condition.

Accounts are created with an account ID, email, login name, and password. There are two account types: admin and customer (or end users). Only cutomers are allowed to create their accounts through the front end of the website; admin accounts are inserted directly into the database. Customer accounts are able to act as both the buyer and seller if they choose. Admin staff users have the ability to generate sales reports.
